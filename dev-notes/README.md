# Notas de Desarrollo

Este directorio agrupa notas, análisis, resúmenes y reportes generados durante el desarrollo del proyecto. **No son documentación oficial** del proyecto — son artefactos de momentos concretos que se conservan como referencia histórica o de trabajo.

> Para la documentación mantenible del proyecto, consulta el directorio [`../docs/`](../docs/README.md).

## ¿Qué hay aquí?

Documentos de un solo propósito o de un momento puntual, que no se mantienen activamente. Típicamente:

- Análisis de problemas o bugs específicos.
- Resúmenes ejecutivos de implementaciones pasadas.
- Checklists de tareas ya completadas.
- Guías rápidas de un solo uso (correcciones puntuales, fixes).
- Reportes críticos de auditorías pasadas.
- Planes de próximos pasos ya ejecutados.
